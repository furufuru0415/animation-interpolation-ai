# アニメーション中割り生成AI（pix2pix × Optical Flow）

## 概要

本プロジェクトでは、アニメーション制作における中割り作業の自動化を目的として、
pix2pixによる画像生成にオプティカルフローを組み合わせた中間フレーム生成手法を開発した。

従来のpix2pix単体では時間的な変化を捉えることが難しく、フレームの重ね合わせのような不自然な画像が生成される問題があった。
本研究では、フレーム間の動き情報（オプティカルフロー）を入力に追加することで、時系列的に一貫性のある画像生成を実現した。

---

## 背景・課題

アニメーション制作は労働集約型の産業であり、特に原画間を補完する「中割り」作業は大きな負担となっている。

しかし、以下の理由から自動化は困難とされている：

* フレーム間の動きが大きい
* 作品ごとに絵柄が異なる
* 時系列的な一貫性が必要

また、pix2pix単体では以下の問題が発生した：

* 中間フレームではなく「前後フレームの重ね合わせ」が生成される
* 動き情報を考慮できていない

---

## 手法

### ① ベースライン（pix2pixのみ）

* 入力：frame t-1, frame t+1（6チャネル）
* 出力：frame t（中間フレーム）

→ 結果：フレームの重ね合わせのような画像が生成される

---

### ② 提案手法（Optical Flow + pix2pix）

フレーム間の動き情報を考慮するため、オプティカルフローを導入

* 入力：

  * frame t-1（3ch）
  * frame t+1（3ch）
  * forward flow（2ch）
  * backward flow（2ch）
    → 合計10チャネル

* 出力：

  * frame t

→ 動き情報を加えることで、時系列的に一貫した画像生成を目指した

## 実験結果

### ■ pix2pixのみ

* フレームの重ね合わせのような画像を生成
* 中間フレームの生成に失敗

### ■ Optical Flowあり

* 中間フレームらしい画像を生成
* 時系列的な一貫性が改善

※結果画像は `results/` フォルダに掲載

## 工夫点

* 入力テンソルの設計（6ch → 10ch）
* forward / backward 両方向のフローを使用
* 損失関数（GAN + L1）のバランス調整
* 識別器と生成器の学習バランスの調整

## まとめ

pix2pix単体では困難であった中間フレーム生成に対し、
オプティカルフローを組み合わせることで改善が見られた。

本手法は、アニメーション制作の効率化に寄与する可能性がある。
